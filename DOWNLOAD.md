# 📥 下载Windows可执行文件

## 🚀 自动构建版本（推荐）

### 方法一：从GitHub Actions下载

1. **访问GitHub仓库**
   - 仓库地址：https://github.com/hayhaps/network-tool

2. **查看构建状态**
   - 点击仓库顶部的 "Actions" 标签
   - 查看最新的构建工作流 "Build Windows Executable"
   - 等待构建完成（通常需要3-5分钟）

3. **下载可执行文件**
   - 点击已完成的构建任务
   - 在页面底部的 "Artifacts" 区域
   - 下载 `network-tool-windows`
   - 解压后得到 `网络故障排查工具.exe`

### 方法二：手动触发构建

1. **访问Actions页面**
   - https://github.com/hayhaps/network-tool/actions

2. **选择工作流**
   - 点击左侧的 "Build Windows Executable"

3. **手动运行**
   - 点击右侧的 "Run workflow" 按钮
   - 选择分支（main）
   - 点击绿色的 "Run workflow" 按钮

4. **等待构建完成**
   - 刷新页面查看构建进度
   - 构建完成后下载artifact

## 📦 手动打包版本

### 在Windows电脑上打包

1. **安装必要软件**
   - Python 3.8+ (https://www.python.org/downloads/)
   - Git (https://git-scm.com/download/win)

2. **克隆仓库**
   ```bash
   git clone https://github.com/hayhaps/network-tool.git
   cd network-tool
   ```

3. **运行打包脚本**
   - 双击 `build_windows.bat`
   - 等待打包完成
   - 在 `dist` 目录找到 `网络故障排查工具.exe`

## 🎯 使用说明

### 首次运行

1. **解压文件**（如果是从GitHub Actions下载的zip）
2. **双击运行** `网络故障排查工具.exe`
3. **允许防火墙访问**（首次运行时Windows会询问）
4. **开始使用**

### 功能模块

#### 1. Ping测试
- 输入目标地址
- 点击"开始Ping"
- 查看连通性测试结果

#### 2. 路由追踪
- 输入目标地址
- 点击"开始路由追踪"
- Windows自动使用tracert命令
- 无需管理员权限

#### 3. DNS查询
- 输入域名或IP地址
- 选择记录类型或自动识别
- 查看解析结果

#### 4. 速度测试
- 点击"开始速度测试"
- 自动识别运营商
- 自动选择最优服务器
- 显示详细测试结果

#### 5. 流量监控
- 点击"开始监控"
- 实时显示上传/下载速度
- 点击"查看网络连接"显示所有连接

#### 6. 网络诊断
- 输入IP列表（每行一个）
- 点击"开始批量Ping"或"开始网络诊断"
- 查看实时测试结果

#### 7. 智能助手
- 输入网络问题
- AI助手基于本地知识库回答

#### 8. 其他功能
- 端口扫描
- IP配置
- Wi-Fi扫描
- 网络拓扑
- 子网计算
- SNMP管理
- VLAN配置

### UI主题

- **工程师主题**（默认）：黑底绿字，终端风格
- **浅色主题**：白底蓝字，现代风格
- **深色主题**：深蓝底粉字，科技风格

点击左下角的"🎨 切换主题"按钮循环切换。

## ⚠️ 注意事项

### Windows Defender误报

某些杀毒软件可能会误报，这是因为：
- PyInstaller打包的程序容易被误报
- 程序需要网络权限

**解决方法**：
1. 添加到杀毒软件白名单
2. 或临时禁用杀毒软件

### 防火墙设置

首次运行时：
1. 允许程序通过Windows防火墙
2. 勾选"专用网络"和"公用网络"

### 管理员权限

大多数功能不需要管理员权限，但：
- 某些网络诊断功能可能需要
- 如遇到权限问题，右键"以管理员身份运行"

## 🔧 常见问题

### 1. 程序无法启动
- 检查是否被杀毒软件拦截
- 尝试以管理员身份运行
- 重新下载最新版本

### 2. 网络功能无法使用
- 检查防火墙设置
- 确保程序有网络访问权限
- 检查网络连接是否正常

### 3. 速度测试失败
- 检查网络连接
- 尝试多次测试
- 检查是否被防火墙拦截

### 4. DNS查询失败
- 检查DNS服务器设置
- 尝试使用其他DNS（如8.8.8.8）
- 检查网络连接

## 📊 系统要求

- **操作系统**：Windows 7/8/10/11
- **内存**：至少2GB RAM
- **磁盘空间**：约100MB
- **网络**：需要网络连接
